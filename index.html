Status

ISHIZENO i8 is currently under development. We are now in the process of gathering requirements and validating some of the core technologies that are planned to be used for the sythesizer, like the Enclustra Mars ZX3 System-on-Module (SoM). If we do not hit any major roadblock, we hope to complete a first prototype this year, and a small production batch next year.

ISHIZENO i8 is designed by Ismael Ghalimi, who is neither a musician nor a synthesizer builder. Instead, he applies his expertise with software development and his fresh perspective on modular synthesis to develop something totally new and radical. Because of that, some of our ideas and designs might be flawed, but we hope that we can quickly converge toward something awesome, thanks to the amazing support that we are receiving from the modular community.

Architecture

Out of the box, the platform only provides limited synthesis capabilities. Instead, these are added through 8 optional submodules (one per stereophonic channel). By default, the Enclustra Mars ZX3 SO-DIMM modules are used both for the core fabric (1 per system) and for the 8 submodules. Programming of the platform and its submodules is done through the Linux operating system.

8 Stereophonic Channels

Each one of the 8 stereophonic channels includes:

  • 1 stereophonic pre-processed digital audio input
  • 1 stereophonic post-processed digital audio output
  • 1 stereophonic raw analog audio output
  • 16 raw analog CV inputs
  • 16 pre-processed digital CV inputs

Just add a good pair of studio monitors...

Digital and Analog Submodules

Up to 8 digital or analog submodules can be added to the tabletop system. Digital submodules can be mounted directly onto the backplane, while analog submodules require the use of individual analog slice boards.

An analog slice is made of an an AD5360 16-channel ADC converter, and a CS42L52 dual-channel DAC converter. Together, they expose all CV inputs as both raw and pre-processed analog signals, while converting the stereophonic analog output of the submodule into a digital signal.

Dynamic Patching

Patching across channels can be done in a traditional way with patch cables, or through the Xilinx Zynq-7020 FPGA. Using dynamic patching, any of the 16 CV inputs and 16 CV outputs can be patched to any of the 16 CV inputs of any channel, or to any voice of its stereophonic audio input.

Additionally, the Xilinx can be used to pre-process any audio or CV I/O, with up to 220 DSP slices. Furthermore, if the Enclustra Mars ZX3 is also used as submodule, an additional 220 DSP slices are available for processing audio and CV I/Os within the submodule itself.

Patching Interface

Dynamic patching can be handled through the tabletop system, through the web user interface, or through a dedicated controller like the ISHIZENO g3, which can be connected to the system over USB or WiFi.

From the tabletop system's faceplate, an input port can be patched to a channel by clicking twice on the port's knob (knob with the same number), then rotating the channel's knob to select the desired CV input. Similar operations can be executed to adjust CV settings or patch output ports.

OLED Displays

The faceplate includes 16 color OLED displays with 16-bit color encoding, 96×64 resolution, and built-in SSD1331 controller. They can display text, geometric shapes, and bitmap images using the Adafruit GFX Library.

These displays are dynamically-assigned to channels, inputs, outputs, or settings, and are designed to be used in conjunction with the rotary encoders for controlling most of the functions of the module and its submodules through the SPI bus of the core Mars ZX3.

Software

ISHIZENO i8 is designed to be as open as possible, across all layers of the software stack:

  • Web user interface implemented using Node.js and AngularJS
  • Mixed signal middleware implemented using C++ and VHDL
  • Online services implemented using the Google Cloud Platform

Since the core Mars ZX3 runs a web server and is used as central nervous system for the system, developing an application for the ISHIZENO i8 is like developing a web application. The user interface is implemented in HTML, JavaScript, and CSS, while integration with external controllers connected to the Zynq over Gigabit Ethernet or WiFi is done with simple web services. This should broaden the community of developers who can create new applications for modular synthesizers.

G3 Grid

The g3 is an optional companion to the i8.
324 keys and 16 pads sensitive to pressure and velocity.
8 customizable controls for mode selection and track transport.


Length: 910mm (35.83")
Width: 215mm (8.36")
Height: 3.14cm (1.24")

While the module can be used with any USB controller, it is being developed jointly with a grid controller, the ISHIZENO g3. This grid is made of 324 keys, 16 drum pads, and 8 controls. Keys, pads, and controls are illuminated by RGB leds and are sensitive to pressure and velocity.

Playing

8 rows of 36 keys for playing across 3 octaves and 8 channels, 1 row for sequencing.

Sequencing

8 rows of 36 keys for multi-channel sequencing, 1 row for note selection.

Mixing

8 rows of 36 keys for channel level setting, 1 row for master output level.

The controller is made of a black walnut frame and is controlled by an embedded Raspberry Pi 2. It also provides a longitudinal groove capable of holding up to 8 tablets in a semi-vertical position, and can be extended with left and right docks for hosting one or two tablets horizontally.

Raison d'être

  • Providing a mixed interface (analog × digital) for processing mixed signals
  • Making it easier to develop custom modules as single circuit printed boards
  • Enhancing the interface of a modular synthesizer with full signal visualization
  • Simplifying the development of rich user interfaces running on any device
  • Experimenting with Low-Frequency Patching (think LFO, but for patching)
  • Using dynamic patching and rotary encoders to record patches and settings

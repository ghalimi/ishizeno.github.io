Status

ISHIZENO 8 is currently under development. We are now in the process of gathering requirements and validating some of the core technologies that are planned to be used for the sythesizer, like the AD75019 analog crosspoint switch. We hope to complete a first prototype for Summer NAMM 2015, and a small production run before the end of the year.

ISHIZENO 8 is designed by Ismael Ghalimi, who is neither a musician nor a synthesizer builder. Instead, he applies his expertise with software development and his fresh perspective on modular synthesis to develop something totally new and radical. Because of that, some of our ideas and designs might be flawed, but we hope that we can quickly converge toward something awesome, thanks to the support that we are receiving from the modular community.

Backplane

Out of the box, the module does not provide any synthesis capabilities. Instead, these are added through optional Arduino shields, with up to 8 shields (one per channel). Programming of the module can be done through the Arduino of any channel, or through the Linux operating system of the module's Raspberry Pi. For example, the Pi could be used as software sequencer. The analog crosspoint switches can also be reconfigured dynamically in order to support low-frequency patching.

8 Polyphonic Channels

Each of the 8 polyphonic channels includes the following:

  • 6 analog ports (A0, A1, A2, A3, A4, A5)
  • 6 knobs
  • 6 digital potentiometers
  • 1 Arduino Uno
  • 1 Arduino shield (optional)

Dynamic Patching

Patching across channels can be done in a traditional way with patch cables, through the analog crosspoint switches, or through multi-channel shields. Using dynamic patching, any pair of ports of any pair of channels can be patched together. Furthermore, dynamic patching allows any analog port to be patched to any potentiometer.

Dynamic patching is enabled by a matrix of 8 or 16 AD75019 analog crosspoint switches. These switches have no gain and a low 150 Ω on resistance. No more than two switches are ever used on the analog path, limiting the on resistance to 300 Ω.

Patching Interface

Dynamic patching can be done through the Eurorack module, the web user interface, or a dedicated controller like the ISHIZENO g3 connected to the Pi through the faceplate's USB 3.1 Type C connector.

From the Eurorack module's faceplate, two ports can be patched by simply clicking on their respective illuminated knobs. When the module is equipped with 8 analog crosspoint switches, 16 ports can be patched together. With 16 switches, this limit is increased to 32.

Features

48-channel Oscilloscope

The 6 analog ports of every polyphonic channel are directly connected to the six analog inputs of the channel's Arduino, which provides 10-bit analog-to-digital converters. This allows the web user interface to be used as a low-resolution 48-channel oscilloscope for signal visualization.

Digital Potentiometers

6 physical knobs are assigned to every polyphonic channel. These knobs use rotary encoders with 24 detents and 24 PPR resolution. They also provide SPST momentary switching and RGB led illumination. Knob settings are passed to a channel through its I²C bus. Settings can be used from a channel's shield using a digital potentiometer like the AD5254.

Knob Patching

All 48 knob settings are converted into analog voltage levels through 12 built-in AD5254 quad digital potentiometers (3 per pair of channels). From there, the voltage level of a channel's knob can be patched to any analog port of the channel, or any of the 16 or 32 analog endpoints available.

Arduino

Every polyphonic channel is controlled by a dedicated Arduino. The channel's analog ports are statically patched to the Arduino's analog inputs. As a result, any standard Arduino shield can be stacked on top of any channel shield, or can be used as channel shield. The Arduinos are directly built into the module's backplane using the Atmel ATMEGA328P microcontroller and the FTDI FT232RL USB interface.

Multi-Channel Shields

By default, a shield is attached to a single channel. Yet some applications might require more than 6 analog ports, more than 6 potentiometers, or more PCB space. For such applications, a multi-channel shield can be used, spanning any number of channels.

Software

ISHIZENO 8 is designed to be as open as possible, across all layers of the software stack:

  • Web user interface implemented using Node.js and famo.us
  • Mixed signal middleware implemented using Arduino
  • Online services implemented using the Google Cloud Platform

Since the Raspberry Pi runs a web server and is used as central nervous system for the module, developing an application for the ISHIZENO 8 is like developing a web application. The user interface is implemented in HTML, JavaScript, and CSS, while integration with the built-in Arduinos and with external controllers connected to the Pi through USB is done with simple web services. This should broaden the community of developers who can create new applications for modular synthesizers.

G3 Grid

The g3 is an optional companion to the i8.
324 keys and 8 drum pads sensitive to pressure, tilt, and velocity.
12 knobs matching the 6 knobs and 6 ports of the i8.


Length: 92cm (36.22")
Width: 20cm (7.87")
Height: 3cm (1.18")

While the module can be used with any USB controller, it is being developed jointly with a grid controller, the ISHIZENO g3. This grid is made of 324 keys, 12 knobs, and 8 drum pads. Keys and drum pads are illuminated by RGB leds and are sensitive to pressure, tilt, and velocity. Knobs use rotary encoders with 24 detents and 24 PPR resolution, as well as SPST momentary switching.

The controller can be used for both playing and sequencing. In playing mode, 8 rows of keys are used for playing, while the top row is used for sequencing. In sequencing mode, 8 rows of keys are used for sequencing, while the top row is used for note selection. The grid supports playing across 3 octaves and 8 channels, as well as 36-step sequencing. When a note is added to a sequence, the 6 potentiometer settings related to the note's channel can be set from the knobs located on the left side of the grid, while the 6 analog levels of the note's channel can be set from the knobs located on the right side.

The controller is made of a black walnut frame and provides a USB connector and a stereo audio output.

Raison d'être

  • Providing a mixed interface (analog × digital) for processing mixed signals
  • Making it easier to develop custom modules as Arduino shields
  • Enhancing the interface of a modular synthesizer with full signal visualization
  • Simplifying the development of rich user interfaces running on any device
  • Experimenting with Low-Frequency Patching (think LFO, but for patching)
  • Using dynamic patching and digital potentiometers to set and record patches
